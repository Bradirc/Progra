Comandos git
Configuración y inicialización del repositorio:
git init: Inicializa un nuevo repositorio Git.
git clone <repositorio>: Clona un repositorio remoto en tu máquina.

Gestión de cambios:
git add <archivo>: Agrega un archivo al área de preparación.
git commit -m "mensaje": Crea un nuevo commit con los cambios en el área de preparación.
git status: Muestra el estado de los archivos en el repositorio.
git diff: Muestra las diferencias entre cambios sin commitear.

Historial y ramas:
git log: Muestra el historial de commits.
git branch: Lista y crea ramas.
git checkout <rama>: Cambia a una rama específica.
git merge <rama>: Fusiona una rama en la rama actual.

Trabajo con remotos:
git remote add <nombre> <URL>: Agrega un nuevo repositorio remoto.
git fetch: Descarga cambios desde un repositorio remoto.
git pull: Descarga y fusiona cambios desde un repositorio remoto.
git push: Sube cambios locales a un repositorio remoto.

Reversión de cambios:
git reset <archivo>: Deshace los cambios en el área de preparación.
git revert <commit>: Crea un nuevo commit que revierte cambios anteriores.
git checkout -- <archivo>: Descarta cambios en el directorio de trabajo.

Almacenamiento temporal de cambios:
git stash: Guarda temporalmente los cambios locales.

Git Ignore:
.gitignore: Archivo para especificar patrones de archivos que Git debe ignorar.

Ayuda y Documentación:
git help <comando>: Muestra la documentación de un comando específico.

Comandos github 

git clone <URL>:
Clona un repositorio remoto en tu máquina local.

git pull origin <rama>:
Descarga cambios desde un repositorio remoto y los fusiona en tu rama local.

git push origin <rama>:
Función: Sube tus cambios locales a un repositorio remoto.

git fetch:
Descarga cambios desde un repositorio remoto, pero no los fusiona automáticamente en tu rama local.

git remote -v:
Muestra los repositorios remotos configurados actualmente.

git branch -a:
Lista todas las ramas locales y remotas.


git merge <rama>:
Fusiona una rama específica en tu rama actual.

git pull request:
Crea una solicitud de extracción (pull request) en GitHub. Este comando a menudo se usa en combinación con otros comandos de Git.

git checkout -b <nueva-rama>:
Crea una nueva rama y cambia a ella.

git fork:
Realiza un fork de un repositorio en GitHub, creando una copia en tu propia cuenta.

git star:
Marca un repositorio como favorito en GitHub.

git follow:
Sigue a un usuario en GitHub.

git issue:
Crea un nuevo informe de problema en GitHub desde la línea de comandos.

git status:
Muestra el estado actual de los archivos en tu repositorio local.
